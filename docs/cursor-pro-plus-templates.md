# Cursor Pro Plus Prompt Templates

이 문서는 Pro Plus 워크플로우 최적화를 위한 프롬프트 스니펫 모음입니다. 파일/경로/의도만 바꿔서 붙여넣어 바로 사용하세요.

---

## 1) 리팩터링 템플릿 (파일/범위 지정)

목표
- [파일/경로]: <예) app/chat/room/[id].tsx>
- 의도: <예) 메시지 렌더 분리 + 성능 개선>

제약
- 타입 안전 유지, 불필요한 try/catch 금지, 기존 스타일/포매터 유지
- 불변성 지키기, 주석은 비필수/핵심만

완료 기준
- read_lints 무오류, 기존 기능/동작 동일
- 변경점 요약과 위험 포인트 설명

출력
- 수정 파일별 diff 형태로 제시 (필요 시 추가 파일 명시)

---

## 2) 버그픽스 템플릿

재현
- 단계: <재현 단계>
- 실제: <실제 결과>
- 기대: <기대 결과>

증거/로그
- 에러/로그: <로그>
- 관련 파일/경로: <파일 목록>

제약
- 기존 퍼블릭 API/디자인 변경 없음, 타입 안전

출력
- 원인 분석 → 수정안 → 변경 코드(diff) → 테스트 포인트

---

## 3) 생성 템플릿 (컴포넌트/훅/스토어)

요구사항
- 목적/역할: <설명>
- API/상태/이벤트: <props/상태/액션>
- UI 구조/상태 전이: <구조>

제약
- 현재 프로젝트 스타일/폴더 구조 준수, 주석 최소

출력
- 파일 경로/코드와 함께 사용법 예시 포함

---

## 4) 전역 교체 미리보기

목표 문자열/정규식: `<패턴>`
범위: `<src 또는 폴더>`
출력: 영향 파일 목록과 줄 미리보기 (적용은 별도 단계)

---

## 5) 컨텍스트 요약 블록 (긴 작업용)

```
[Context Summary]
- 변경 대상: <파일/모듈>
- 핵심 규칙: <스타일/제약>
- 완료 기준: <빌드/테스트/린트>
```

이 블록을 프롬프트 상단에 유지해 세션 길어져도 안정적으로 문맥을 유지합니다.
